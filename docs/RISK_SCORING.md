# ⚠️ Algorithmic Risk Scoring & XAI

## Triage Index

Each patient is assigned a **risk score** (0–100) and a corresponding **care level**:

| Risk Score | Care Level | Action |
|------------|------------|--------|
| 0–30 | `HOME_CARE` | Self‑care advice, follow‑up in 24h |
| 31–70 | `CLINIC_VISIT` | Schedule outpatient appointment |
| 71–100 | `EMERGENCY_ROOM` | Immediate escalation, telemedicine routing |

The score is derived from the LLM’s structured output plus deterministic overrides (see below).

## Deterministic Safety Rails

Because LLMs can hallucinate, we implement a `gatekeeper.py` that scans the raw user input for emergency keywords:

- `chest pain`
- `difficulty breathing`
- `severe bleeding`
- `unconscious`
- `stroke symptoms`

If any match, the system **bypasses the LLM entirely** and sets `care_level = EMERGENCY_ROOM` and `risk_score = 100`.

## Explainable AI (XAI) for Clinicians

The doctor dashboard includes an **XAI Card** that shows:

- The original clinical reasoning text generated by the LLM.
- High‑risk tokens (e.g., “chest pain”, “stroke”) highlighted in red with a pulsating badge.
- The algorithmic severity index as a progress bar.

This transparency builds trust and allows clinicians to audit the AI’s decision.
